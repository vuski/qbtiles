# 벤치마크 결과: PMTiles vs QBTiles 인덱스 크기 비교

## 관찰

- 12개 파일 중 10개에서 QBTiles가 더 작음
- 엔트리 수가 많고 공간적으로 밀집된 데이터일수록 QBTiles 이점이 큼
- hillshade (462만 엔트리, 밀집 래스터): **47.8% 절감**
- 전세계 OSM (1.6억 엔트리): **21.8% 절감** (gzip), **33.4% 절감** (raw)
- 압축 전 엔트리당 **2.15 bytes 절약** (6.43 → 4.28) — tile_id delta를 저장하지 않는 효과
- gzip 압축 후 차이가 줄어드는 이유: PMTiles의 delta 패턴도 gzip에 의해 잘 압축됨
- QBTiles가 더 큰 경우: building (+2.3%), osm_z0-10 (+5.5%) — 다중 줌레벨 혼합 또는 공간적으로 분산된 경우

## 조건

- PMTiles: tile_id 기준 정렬, serialize_directory() (varint delta 인코딩, gzip)
- QBTiles: quadkey 기준 정렬, write_tree_bitmask_to_single_file() (비트마스크 BFS + 열방향 varint, gzip)
