# 쿼드키 Int64 인코딩

## 표준 쿼드키

지도 타일은 줌레벨마다 4분할된다. 각 분할에 0-3 숫자를 부여하면 타일의 위치를 문자열로 표현할 수 있다.

```
Level 1:  4개 타일   (0, 1, 2, 3)
Level 2:  16개 타일  (00, 01, 02, 03, 10, 11, ..., 33)
Level 3:  64개 타일  (000, 001, ..., 333)
...
Level z:  4^z 개 타일
```

각 자릿수는 2×2 분할의 위치를 나타낸다: 0=좌상, 1=우상, 2=좌하, 3=우하.

각 자릿수의 비트 인코딩:

```
digit = ((y >> i) & 1) << 1 | ((x >> i) & 1)

  digit 0: x=0, y=0 (좌상)
  digit 1: x=1, y=0 (우상)
  digit 2: x=0, y=1 (좌하)
  digit 3: x=1, y=1 (우하)
```

## 문자열의 한계

쿼드키를 정수로 다루면 정렬/비교가 빠르지만, 문자열→정수 변환에 모호성이 있다:

```
"0"   → 0
"00"  → 0   ← 구분 불가!
"000" → 0   ← 구분 불가!
"032" → 32
"0032"→ 32  ← 구분 불가!
```

줌레벨이 다른 타일인데 같은 정수가 되어버린다.

## 0b11 프리픽스 트릭

해결책: 쿼드키 앞에 **`3`(이진수 `11`)을 prefix**로 붙인다.

```
"0"   → "30"    → 이진: 11 00             → 정수 12
"00"  → "300"   → 이진: 11 00 00          → 정수 48
"000" → "3000"  → 이진: 11 00 00 00       → 정수 192
"032" → "3032"  → 이진: 11 00 11 10       → 정수 206
```

모두 다른 정수가 된다.

### 왜 `3`인가?

쿼드키 각 자릿수는 0, 1, 2, 3 중 하나다. `3`은 유효한 자릿수이지만, **첫 번째 위치에서는 prefix 표식으로만 사용**된다. 이진수로 `11`이므로 2비트 쌍을 스캔할 때 `11`을 만나면 "여기서부터 실제 쿼드키 시작"이라는 뜻이다.

## 변환 함수

### z/x/y → int64

```python
def tile_to_quadkey_int64(z, x, y):
    quadkey_int64 = 3  # 프리픽스 0b11
    for i in reversed(range(z)):
        digit = ((y >> i) & 1) << 1 | ((x >> i) & 1)
        quadkey_int64 = (quadkey_int64 << 2) | digit
    return quadkey_int64
```

### int64 → z/x/y

```python
def quadkey_int64_to_zxy(qint64):
    x = y = 0
    found_prefix = False
    z = 0
    for shift in reversed(range(0, 64, 2)):
        digit = (qint64 >> shift) & 0b11
        if not found_prefix:
            if digit == 0b11:
                found_prefix = True
            continue
        x = (x << 1) | (digit & 1)
        y = (y << 1) | ((digit >> 1) & 1)
        z += 1
    return z, x, y
```

## 속성

- **정렬 가능**: int64 쿼드키를 정렬하면 공간적 인접성이 유지된다
- **컴팩트**: 가변 길이 문자열 대신 고정 64비트 정수
- **줌레벨 범위**: 64비트 - 2비트(프리픽스) = 62비트 / 2비트(레벨당) = **최대 줌 31**
- **빠른 비교**: 정수 연산으로 부모-자식 관계를 판별할 수 있다
  - 부모 quadkey = 자식 quadkey >> 2
  - 자식 quadkey = (부모 quadkey << 2) | digit
